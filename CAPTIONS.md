# Captions Endpoint

The wrapper serves random captions for the JUCE dice button via `GET /captions`.

## Endpoints

| Endpoint | Description |
|----------|-------------|
| `GET /captions` | Random caption from the default pool (ACE-Step examples) |
| `GET /captions?lora=koan` | Random caption from a LoRA-specific pool |
| `GET /captions/pools` | List available pools and their sizes |

## Response format

```json
{
  "caption": "An energetic glitch-hop track...",
  "pool": "koan",
  "pool_size": 42
}
```

## How captions.json is built

`captions.json` is generated by `build_captions.py`. It reads:

- **LoRA pools**: sidecar `.txt` files from training directories (the `caption:` line)
- **Default pool**: ACE-Step `examples/text2music/*.json` files (the `caption` field)

### Regenerate captions.json

```bash
cd ~/gary-backend-spark/ace-step-wrapper

python3 build_captions.py \
  --lora koan:/home/kev/saos_training \
  --default /home/kev/ace/ACE-Step-1.5/examples/text2music \
  -o captions.json
```

### Add a new LoRA pool

Append another `--lora name:/path/to/sidecars`:

```bash
python3 build_captions.py \
  --lora koan:/home/kev/saos_training \
  --lora jesse:/path/to/jesse/training/sidecars \
  --default /home/kev/ace/ACE-Step-1.5/examples/text2music \
  -o captions.json
```

### Deploy changes

`captions.json` is volume-mounted into the wrapper container (read-only). After regenerating, restart the wrapper to pick up the new file:

```bash
cd ~/gary-backend-spark
docker compose restart ace-step-wrapper
```

No image rebuild needed.

## Sidecar format reference

The training sidecars are key-value `.txt` files produced by `understand_music` / the ACE-Step 4B LM:

```
caption: An intense, cinematic synthwave track...
genre: synthwave, cinematic
bpm: 128
key: F minor
signature: 4
is_instrumental: true
custom_tag: patch
lyrics:
[Instrumental]
```

Only the `caption:` line is extracted. BPM, key, duration, and lyrics are set by the user in the DAW.
